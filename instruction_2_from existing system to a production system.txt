      <h1>Claude Prompt — MENA News Intelligence System v2</h1>
      <p class="muted">Use this prompt to help Claude directly implement the modular refactor.</p>

      <h2>Project Goal</h2>
      <p>
        Build a modular Python system that refactors the current RSS → Firestore → scoring pipeline into four clearly separated layers:
      </p>
      <ol>
        <li>Raw news collection</li>
        <li>Independent scoring engine</li>
        <li>Telegram daily review and human selection</li>
        <li>Weekly report generation + email delivery</li>
      </ol>

      <h2>Current Situation</h2>

      <h3>Existing Collection</h3>
      <ul>
        <li>Reads around 21 RSS feeds</li>
        <li>Uses <code>feedparser</code></li>
        <li>Extracts <code>title</code>, <code>url</code>, <code>description</code>, <code>published_date</code></li>
        <li>Deduplicates by URL</li>
        <li>Stores into Firestore collection <code>news</code></li>
      </ul>

      <h3>Existing Scoring</h3>
      <p>Two scores already exist:</p>
      <ul>
        <li><code>relevance_score</code></li>
        <li><code>importance_score</code></li>
      </ul>

      <pre><code>total_score = 0.65 * relevance_score + 0.35 * importance_score</code></pre>

      <h3>Current Problem</h3>
      <ul>
        <li>Daily digest sends all past-24h articles</li>
        <li>Scoring is computed but not properly used to filter/rank daily candidates</li>
        <li>No proper human selection workflow</li>
        <li>No structured weekly output for sections 四 / 五 / 九</li>
        <li>No modular separation between raw data, scoring, human selection, and reporting</li>
      </ul>

      <h2>Required Workflow</h2>

      <h3>Daily Workflow</h3>
      <ol>
        <li>Collect raw news from RSS feeds</li>
        <li>Score raw news independently</li>
        <li>Push only candidate news to Telegram</li>
        <li>Telegram buttons let me assign each article to:
          <ul>
            <li>选四</li>
            <li>选四⭐</li>
            <li>选五</li>
            <li>选五⭐</li>
            <li>四五都选</li>
            <li>忽略</li>
          </ul>
        </li>
        <li>Human selection must override machine ranking</li>
      </ol>

      <h3>Weekly Workflow</h3>
      <ol>
        <li>Read selected items for current week</li>
        <li>Group selected items into 四 and 五</li>
        <li>Generate:
          <ul>
            <li>四. 募资市场动态</li>
            <li>五. 投资者关注</li>
            <li>九. 募资的思考和反思</li>
          </ul>
        </li>
        <li>Save weekly report to Firestore</li>
        <li>Email the final report to me</li>
      </ol>

      <h2>Core Principles</h2>
      <ol>
        <li>Raw news must remain independent from scoring and selection</li>
        <li>Scoring must be separated so weights can be changed later</li>
        <li>Human selection is the strongest signal</li>
        <li>Telegram is for daily review</li>
        <li>Email is for final weekly delivery</li>
        <li>My Telegram click is not just “select”; it is also section routing</li>
      </ol>

      <h2>Daily Telegram Review Requirements</h2>

      <h3>Daily Push Logic</h3>
